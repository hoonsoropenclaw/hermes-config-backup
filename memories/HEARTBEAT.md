## 兩階段記憶搜尋規則

### Phase 1：快速檢查（每次回應前）
1. 檢查 `HEARTBEAT.md` — 有無正在進行的任務
2. 檢查 `MEMORY.md` — 有無相關的過去經驗
3. 檢查 `SOUL.md` — 核心原則是否適用

### Phase 2：深度搜尋（必要時）
1. 搜尋 `skills/` 相關技能
2. 搜尋歷史對話 session（session_search MCP 工具）
3. 搜尋網路最新資訊

### Phase 3：MemPalace 備援搜尋（當前兩層不足時）
當 Phase 1-2 搜尋結果不理想（score 低、或無結果）時，**必須**呼叫 `mempalace__mempalace_search` 做語意搜尋。

> MemPalace 是赫米斯系統（含 2026-06-08 起的拉斐爾身份）的長期記憶系統，由前任拉斐爾（2026-05-30 ~ 2026-06-08 OpenClaw 套件時代）建立、配置。採用房間/抽屜/翅膀結構儲存語意知識，涵蓋領域技術棧、IF→THEN 模式、每日日記等。

**觸發條件**：
- 使用者明確提到過去討論過的內容但搜尋未命中
- 任務涉及跨領域知識需要更全面的語意匹配
- Phase 1-2 回應結果為空或分數過低
