# 學校 HR 招聘自動化參考

## 核心痛點：代理教師（即代課）填補

學校 HR 與企業 HR 最大的差異在於**代理教師管理的即時性**：
- 老師突然請假/離職 → 需要在 24-48 小時內找到代理教師
- 企業 ATS（Greenhouse/Lever）週期太長，不適用
- 每次只填補 1-3 人，規模太小

### 即時填補流程

```
1. 確認需求（科目、年級、期間）
2. 查詢候補名單（資料庫）
3. 電話/訊息聯繫
4. 確認意願與可上班時間
5. 發送 offer（口頭 → 書面）
6. 資料繳交（教師證影本、良民證等）
7. 報到
```

### LLM 應用優先順序

| 優先級 | 任務 | 說明 |
|--------|------|------|
| 1 | 生成職缺描述 | 快速發佈招募，格式統一 |
| 2 | 履歷初篩 | 用 LLM 快速識別符合條件者 |
| 3 | 生成面試問題 | 依科目和年級客製化 |
| 4 | 候選人溝通 | 個人化 email/訊息 |
| 5 | 入職文件檢查 | 確認文件完整性 |

## 學校 HR 法規參考要點

- **代理教師資格**：須具備教師證，或符合「代理教師辦法」之暫緩條件
- **聘期計算**：按實際上班日計算，不满一個月按比例
- **鐘點費上限**：各縣市有公告標準，不可超出上限
- **資料繳交**：教師證正本查驗、良民證（三個月內）、體檢表

**If** 需要自動化學校 HR 任務，**Then** 先用 `anthropic-compliance-check` 確認法規範圍，再設計自動化環節
