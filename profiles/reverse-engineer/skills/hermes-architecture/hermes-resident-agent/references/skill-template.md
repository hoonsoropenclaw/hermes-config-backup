# 專屬 Skill 的 SKILL.md 範本

常駐代理的專屬 skill 結構化這個代理的工作流程。`persona.md` 定義「**是誰**」，專屬 skill 定義「**怎麼做**」。

## 結構（6 段）

```markdown
---
name: <skill-name>
description: <一句話描述這個 skill 做什麼>
version: 1.0.0
author: <誰建的>
tags: [<分類 tag>]
---

# <Skill 名稱> Skill

<一句話開場，這個 skill 的核心目標>。

## 觸發情境

- <情境 1>：例如「使用者提出新產品構想」
- <情境 2>：例如「使用者要求做競品分析」
- <情境 3>：例如「進入 PRD 流程前的前期調研」

## 標準流程（<N> 步）

### 1. <步驟名>
- <這個步驟做什麼>
- <必填項目、檢核表>

### 2. <步驟名>
- ...

...（共 5-6 步）

### N. <交付步驟>
- <交付物格式>
- <命名慣例>
- <結構定義>

## 必用工具

- `<tool>`：<用這個工具做什麼>
- `<tool>`：<用途>
- `<tool>`：<用途>

## 品質檢核（自我審查清單）

產出前自問：
1. <檢核問題 1>
2. <檢核問題 2>
3. <檢核問題 3>
4. <檢核問題 4>
5. <檢核問題 5>

任一為否，回去補。

## 詳見
- `references/<相關檔>.md` — <這個 references 檔的內容>
```

## 實例：市場研究 skill

```markdown
---
name: market-research
description: 市場調研方法論：標竿分析、SWOT、Persona、TAM/SAM/SOM、風險盤點。載入此技能後代理會自動套用結構化調研流程。
version: 1.0.0
author: hoonsor
tags: [research, market, strategy, swot, persona, benchmark]
---

# Market Research Skill

市場調研方法論技能，給市場策略代理使用。

## 觸發情境

- 使用者提出新的產品構想
- 使用者要求做競品分析 / 標竿研究
- 使用者要進入 PRD 流程前的前期調研

## 標準流程（6 步）

### 1. 釐清問題
- 反問 3-5 個關鍵問題
- 取得使用者確認後再進入下一步

### 2. 標竿掃描
- 直接競品 3-5 個、間接 2-3 個、跨領域典範 1-2 個
- 必填欄位：定價、核心功能、差異化、評分、最近更新、來源 URL

...（共 6 步）

## 必用工具

- `web_search` / `web_extract`：找競品資訊、使用者評論
- `arxiv`：找相關學術論文
- `browser`：抓動態網頁（App Store 評論、G2 評分）
- `write_file` / `read_file` / `patch`：管理報告檔案

## 品質檢核（自我審查清單）

產出前自問：
1. 每個論斷都有引用嗎？
2. Persona 寫得像真的人嗎？
3. TAM 數字有標假設嗎？
4. SWOT 真的是從資料推出，不是套模板？
5. 第 8 節「建議定位」具體到 product-planner 可以直接動工？

## 詳見
- `references/benchmark-checklist.md` — 標竿掃描詳細檢核表
```

## 寫 skill 的常見錯誤

1. **frontmatter 缺 name/description**：代理不會自動載入
2. **觸發情境寫太籠統**：「使用者有需求時」太廣，改成具體情境
3. **流程不夠具體**：每步要有「做什麼」+「產出什麼」
4. **必用工具沒列**：代理會亂用其他不相干的工具
5. **缺自我審查清單**：這是品質把關的最後一關，不能省

## skill 跟 persona 的分工

| 檔案 | 回答的問題 | 比喻 |
|------|----------|------|
| persona.md | 這個代理是誰？有什麼信念？ | 人的性格、價值觀 |
| skills/<name>/SKILL.md | 這個代理接到任務後怎麼做？ | 人的 SOP、專業方法論 |
| references/*.md | 詳細的參考資料、檢核表 | 人的工具書、筆記 |

## 撰寫順序建議

1. 先寫 persona.md（確定身份、信念、邊界）
2. 再列 1-3 個這個代理會反覆做的任務類型
3. 每個任務類型抽出一個 skill（標準流程、必用工具、檢核清單）
4. skill 內的詳細資料（檢核表、範本）放 `references/`
