# SKILL.md - automation-workflows 自動化工作流設計

## 身份
- **name**: automation-workflows
- **description**: 為獨工作者（solopreneurs）識別、設計和實施無代碼自動化工作流的結構化 playbook。涵蓋從審計耗時任務到建構、測試和維護自動化的完整週期。包括 ROI 計算以優先考慮哪些自動化值得建構。

## 核心概念

### 為何需要自動化？
- 將時間回收用於更高價值的任務
- 減少人為錯誤
- 標準化流程
- 提高生產力

### 自動化類型
1. **觸發型**: 事件觸發動作（表單提交 → CRM 新增 → 發送歡迎郵件）
2. **排程型**: 定期執行任務（每週監控、每月報告）
3. **條件型**: 滿足條件時執行分支流程

## 工具生態

### 主要平台
| 平台 | 費用 | 特點 |
|------|------|------|
| **Zapier** | 免費/起 $20/月 | 最成熟，支援 5000+ 應用 |
| **Make** | 免費/起 $9/月 | 視覺化流程編輯，彈性高 |
| **n8n** | 免費/自託管 | 開源，可自行部署 |

### 選擇原則
- 選 Zapier：需要穩定、廣泛應用整合
- 選 Make：需要複雜邏輯、可視化編輯
- 選 n8n：需要自託管、成本控制、客製化

## 工作流識別框架

### ROI 計算公式
```
ROI = (年節省時間 × 每小時價值) / 建構成本
Payback Period = 建構成本 / (月節省價值)
```

### 優先級標準
- ✅ **立即建構**: Payback Period < 3 個月
- ⚠️ **考慮建構**: Payback Period 3-6 個月
- ❌ **暫時跳過**: Payback Period > 6 個月

### 審計問題清單
1. 什麼任務最耗費時間？
2. 哪些是重複性任務（每週/每月）？
3. 哪些任務是手動複製粘貼？
4. 哪些流程有多個步驟但很少變化？
5. 如果能自動化，你最希望騰出多少時間？

## 工作流設計模式

### 常見模式

#### 1. 表單驅動 CRM 同步
```
用戶提交表單 → Zapier/Make 觸發 → 建立 CRM 記錄 → 發送確認郵件
```

#### 2. 發票自動化
```
付款確認 → 生成發票 → 發送給客戶 → 更新財務記錄
```

#### 3. 社群發布流水線
```
部落格發布 → 生成摘要 → 排程社群發布 → 通知訂閱者
```

#### 4. 客戶健康監控
```
每週資料收集 → 計算健康分數 → 低於閾值 → 發送跟進郵件
```

#### 5. 入職序列
```
合約簽署 → 觸發入職流程 → 發送歡迎郵件 → 安排介紹會議
```

## 建構步驟

### 步驟 1：識別任務
```markdown
## 任務審計
1. 列出所有耗時任務
2. 估計每週/每月頻率
3. 評估複雜度（簡單/中等/複雜）
4. 計算潛在節省時間
```

### 步驟 2：選擇工具
根據以下因素選擇：
- 應用整合需求
- 預算限制
- 技術能力
- 數據敏感性

### 步驟 3：設計流程圖
```
[觸發事件] → [步驟1] → [條件判斷] → [步驟2a/2b] → [動作]
```

### 步驟 4：建構原型
1. 從最簡單的流程開始
2. 先測試單一路徑
3. 逐步添加條件分支
4. 記錄每個步驟的輸入/輸出

### 步驟 5：測試與驗證
- 使用測試資料完整運行
- 驗證所有分支路徑
- 檢查錯誤處理
- 確認通知/日誌功能

### 步驟 6：部署與監控
- 設定監控儀表板
- 建立失敗警報
- 定期檢視優化

## 維護最佳實踐

### 季度審查
1. 哪些工作流不再使用？
2. 哪些可以合并或簡化？
3. 哪些需要升級或重構？

### 文檔記錄
- 流程圖（PNG/Mermaid）
- 連接的應用和帳號
- 潛在故障點
- 緊急關閉步驟

### 錯誤應對
- 設置電子郵件警報
- 準備手動備份流程
- 定期備份配置

## 使用時機

### 適合自動化的場景
- ✅ 重複性任務（每週/每月）
- ✅ 多步驟但規則明確的流程
- ✅ 時間敏感的動作
- ✅ 跨多個應用的數據傳遞

### 不適合自動化的場景
- ❌ 需要人類判斷的決策
- ❌ 極少運行的例外流程
- ❌ 經常變化的流程
- ❌ 涉及敏感數據的高風險操作

## ROI 範例

### 案例：社群發布自動化
- **建構時間**: 2 小時
- **工具成本**: $0（免費方案）
- **每週節省**: 1 小時
- **月節省**: 4 小時
- **年薪價值**（假設 $50/小時）: $2,400
- **ROI**: ∞（無建構成本）

### 案例：CRM 同步自動化
- **建構時間**: 4 小時
- **工具成本**: $20/月（Zapier Basic）
- **每週節省**: 2 小時
- **月節省**: 8 小時
- **年薪價值**（假設 $50/小時）: $4,800
- **Payback Period**: 2 個月
