# Portal Judge Agent — 評價報告模板

> 子代理跑完評價後，**用這個格式**回報給主 session。不要自由發揮。

```
=== 評價完成 ===
Work: <title> (id: <work_id>)
URL: <evaluated url>
評審: portal-judge-agent-v1
評價時間: <ISO 8601, e.g. 2026-06-07T22:30:00+08:00>
評價耗時: <X 秒>

評分（三維度各 1-10）：
- 設計感 (score_design): X/10 — <一句話理由，提到具體元素>
- 實用性 (score_practical): Y/10 — <一句話理由，提到具體場景>
- 直覺性 (score_intuitive): Z/10 — <一句話理由，提到具體動作>
- 平均: (X+Y+Z)/3 = N.N/10

互動紀錄：
- 互動的 tab / 區塊: <列出>
- 未互動的區塊: <列出，並標明「未測試」>
- 截圖: <列出所有截圖路徑>

Feedback 全文（200-400 字繁體中文）：
---
優點：
- [具體功能/設計] 在 [場景] 下表現 [好/快/直覺]
- [具體元素] 處理得 [為什麼好]

問題：
- 在 [場景] 下，[動作] 後 [症狀]（附 console error 或截圖路徑）
- [頁面/區塊] 對 [裝置/使用者] 不友善，因為 [原因]

建議（可執行）：
- 如果把 [A] 改成 [B]，預期 [直覺性] 從 X 分提升到 Y 分
- 修復 [bug] 後，建議重新跑一次評分
---

驗證：
- POST 狀態: <201 / 失敗原因 / "skipped per user instruction">
- DB 確認: <"evaluations 表有 1 筆新紀錄" / "not applicable">
- Console error 數: <N>（每個都列訊息）
- 主 session 親自驗證項目: <列出主 session 怎麼驗證這份報告>
- 無法親自驗證項目: <例如 "視覺細節 — vision 模型未配置">

AI 評審的自覺聲明：
- 我不是 <設計師/工程師/文案>，評分有 [主觀/邊界] 性
- 這個分數是 [客觀基準線/初步篩選]，使用者的評分優先
- 如果使用者的評分跟我差 > 3 分，請告訴我哪裡我想錯了
```

## 必填項目檢查表

子代理在送出回報前**自我檢查**：

- [ ] 三個分數都有**一句話以上的具體理由**（不是空泛「好」「差」）
- [ ] Feedback 至少 200 字（不達 200 字代表看不夠仔細）
- [ ] 至少有 1 條「可執行建議」（不是「改進」「優化」這種空話）
- [ ] 列出了互動過的 tab 跟未互動的 tab
- [ ] Console error 數字具體（不是「有 console error」）
- [ ] 標出哪些項目主 session **無法**親自驗證（特別是視覺、AI 評審限制）

## 不接受的回報模式

❌ 自由散文：「我覺得這個作品還不錯，整體而言...」
❌ 只打分沒理由：「設計 7、實用 8、直覺 7」沒附文字
❌ 預設中央傾向（全部 5-7 分、沒有 < 5 或 > 8）
❌ 沒互動就打分（首頁看一下就寫「直覺性 8/10」）
❌ 「我無法判斷」規避評分（你是 AI 評審、給分是你的職責、給不出來就給 5 + 解釋為什麼保守）
